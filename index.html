Week 5: Cascading Style Sheets (CSS) II

Published on November 02, 2022

In week 5 I finished lab tutorial week 5, in which I learnt to:

  1. Become more familiar with CSS, more specifically styling text. For example, drop caps.
  2. Understand the importance of specifying the font-family with a fallback, so that the webpage will display another font in the absence of the original and intended one.
  3. Use web fonts, such as Google Fonts.
  4. Style paragraphs such as their: line-height, margin, text-indent, dropcap, text align.
  5. Float an image and wrap text around it.

One key point of Week 5 was to understand the CSS box model, which refers to the space taken by HTML elements. For my independent studies, in order to gain a better understanding of the box model I read MDNs Introduction to the CSS box model which offered more insight into the four individual parts of the box model such as the Content area, Padding area, Border area and Margin area. Each with their respective edges and values. I also learnt about the relative and obsolute length units, including Pixels (px(picture x element)), percentage (%), rem, em, vw and vh. For now, I will primarily be using percentage as it will be useful when I proceed onto making my webpage responsive. Furthermore, after further research I discovered the use of the "parallax" effect that can be used in Web and Game Development, this effect gives off the illusion of making something seem further away by having it move slower in relation to the main body as the user navigates throughout the webpage. This is something I plan on implementing in the near future. For more information regarding the background on web pages I also read MDNs CSS Backgrounds and Borders.

I took notice that my previous style was not exactly eye-candy, for example the contrast between the blue text and the background made it a somewhat nuisance to read, so I have resorted to a more formal colour-palette. Also, before I understood how to float an image and to have text wrap around it, the W3S image just "sat" underneath the text. Here is a before and after image after floating the W3S image.

Week 6: Responsive Web Development (RWD)

Published on November 09, 2022

In week 6 I started making my webpage responsive, meaning the webpage is tailored to the size, orientation and dimensions of any device. I first sketeched various wireframes of different layouts devices and taking their screen sizes into account(see below). I then set up the responsive grid by using grid-area for each appropiate HTML elements (e.g. webpagetitle,menu, nav, table, main, aside, footer and added media queries to set up 1 column (mobile devices), 2 column (tablet) and 3 (widescreen) layouts. After this I styled each screen width, for example the timetable will not show up on devices with a width smaller than 35em. In short, this week I learnt to:

  1. Sketch wireframes of different screen dimensions.
  2. Set up the responive grid.
  3. Style the webpage for different screen sizes.
  4. Hide elements, for example the timetable.

This week I created an original tutorial for "Establishing Effective Study Habits" for the linked tutorial webpage. For my independent studies I discovered the use of keyframes. With this, I was able to animate the background colour of the banner as it goes through different colours and have it loop. I also used keyframes to move a procedural texture I made in Blender in the background as an image. To better understand media queiries and making webpages responsive I read MDNs Media Queries.
